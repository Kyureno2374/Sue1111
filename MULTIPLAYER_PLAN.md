# План реализации мультиплеера для игры крестики-нолики

## Обзор
Текущая игра работает только против ИИ. Нужно добавить возможность игры между реальными игроками в реальном времени.

## Текущее состояние
- ✅ Игра против ИИ работает
- ✅ Система ставок и баланса
- ✅ Админка для управления играми
- ✅ Лидерборд
- ✅ Профиль пользователя
- ✅ Система уведомлений

## План реализации мультиплеера

### 1. Система поиска игроков (Lobby)

#### 1.1 Создание лобби
- **Файл**: `components/lobby-screen.tsx` (новый)
- **Функционал**:
  - Показ доступных игр для присоединения
  - Создание новой игры с настройкой ставки
  - Фильтрация игр по ставке, статусу
  - Автообновление списка игр

#### 1.2 API для лобби
- **Файл**: `app/api/games/lobby/route.ts` (новый)
- **Эндпоинты**:
  - `GET /api/games/lobby` - получить список доступных игр
  - `POST /api/games/lobby` - создать новую игру
  - `PUT /api/games/lobby/[id]/join` - присоединиться к игре

### 2. WebSocket интеграция

#### 2.1 Настройка WebSocket сервера
- **Файл**: `lib/socket-server.ts` (новый)
- **Технология**: Socket.IO или нативный WebSocket
- **Функционал**:
  - Подключение игроков к комнатам
  - Передача ходов в реальном времени
  - Уведомления о событиях игры

#### 2.2 WebSocket клиент
- **Файл**: `lib/socket-client.ts` (новый)
- **Интеграция**: с существующим `use-multiplayer.tsx`
- **События**:
  - `game:join` - присоединиться к игре
  - `game:move` - сделать ход
  - `game:update` - обновление состояния игры
  - `game:end` - завершение игры

### 3. Обновление игровой логики

#### 3.1 Модификация API игры
- **Файл**: `app/api/games/[id]/move/route.ts`
- **Изменения**:
  - Добавить проверку, что ход делает правильный игрок
  - Добавить валидацию для мультиплеера
  - Отправка WebSocket уведомлений после хода

#### 3.2 Обновление компонента игры
- **Файл**: `components/game-board.tsx`
- **Изменения**:
  - Добавить индикатор чей ход

### 4. Система комнат и матчмейкинг

#### 4.1 Управление комнатами
- **Файл**: `lib/game-rooms.ts` (новый)
- **Функционал**:
  - Создание комнат для каждой игры
  - Подключение игроков к комнатам
  - Отслеживание состояния игроков

#### 4.2 Матчмейкинг
- **Логика**:
  - Игрок создает игру → ждет второго игрока
  - Игрок присоединяется к существующей игре
  - Автоматическое начало игры при подключении второго игрока
