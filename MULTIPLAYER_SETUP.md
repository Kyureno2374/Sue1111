# Настройка мультиплеера для игры крестики-нолики

## Обзор
Мультиплеер позволяет игрокам сражаться друг с другом в реальном времени через WebSocket соединения.

## Предварительные требования

### 1. Установленные зависимости
```bash
npm install socket.io socket.io-client
```

### 2. Обновление базы данных
Выполните миграцию для добавления полей мультиплеера:

```bash
# Подключитесь к вашей базе данных Supabase
# Выполните SQL скрипт из файла migration-multiplayer.sql
```

### 3. Переменные окружения
Добавьте в ваш `.env.local`:

```env
NEXT_PUBLIC_SOCKET_URL=http://localhost:3000
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

## Структура файлов

### WebSocket сервер
- `lib/socket-server.ts` - WebSocket сервер для обработки соединений
- `lib/socket-client.ts` - WebSocket клиент для фронтенда

### API для мультиплеера
- `app/api/games/lobby/route.ts` - API для лобби (поиск и создание игр)

### Компоненты интерфейса
- `components/lobby-screen.tsx` - Экран лобби для поиска игр
- Обновленный `components/home-screen.tsx` - Выбор режима игры
- Обновленный `components/user-interface.tsx` - Интеграция мультиплеера

## Запуск мультиплеера

### 1. Инициализация WebSocket сервера
В вашем `next.config.mjs` или серверном файле добавьте:

```javascript
import { initializeSocketServer } from '@/lib/socket-server'

// В функции создания сервера
const server = createServer(app)
initializeSocketServer(server)
```

### 2. Запуск приложения
```bash
npm run dev
```

## Функциональность

### Создание игры
1. На главном экране выберите "vs Player"
2. Введите ставку
3. Игра создается и ждет второго игрока

### Поиск игры
1. Перейдите в лобби (кнопка "Find Match")
2. Выберите доступную игру
3. Нажмите "Join Game"

### Игровой процесс
- Ходы передаются в реальном времени
- Автоматическое определение победителя
- Обновление баланса после завершения

## События WebSocket

### Клиент → Сервер
- `game:join` - Присоединиться к игре
- `game:move` - Сделать ход
- `ping` - Проверка соединения

### Сервер → Клиент
- `game:player_joined` - Игрок присоединился
- `game:move_made` - Ход сделан
- `game:player_disconnected` - Игрок отключился
- `game:error` - Ошибка
- `pong` - Ответ на ping

## Безопасность

### Валидация
- Проверка баланса перед созданием игры
- Валидация ходов на сервере
- Проверка прав доступа к игре

### Обработка отключений
- Автоматическое удаление старых игр
- Возврат ставок при технических проблемах
- Таймауты для неактивных игроков

## Мониторинг

### Логирование
Все события логируются через `lib/utils/logger.ts`:
- Подключения/отключения
- Создание/присоединение к играм
- Ходы и ошибки

### Метрики
- Количество активных игр
- Время ожидания в лобби
- Статистика побед/поражений

## Тестирование

### Локальное тестирование
1. Откройте приложение в двух вкладках
2. Создайте игру в одной вкладке
3. Присоединитесь к игре во второй вкладке
4. Проверьте игровой процесс

### Тестирование WebSocket
```javascript
// В консоли браузера
const socket = io('http://localhost:3000')
socket.emit('game:join', { gameId: 'test', userId: 'user1', username: 'TestUser' })
```

## Устранение неполадок

### Проблемы с подключением
1. Проверьте `NEXT_PUBLIC_SOCKET_URL`
2. Убедитесь, что WebSocket сервер запущен
3. Проверьте CORS настройки

### Проблемы с базой данных
1. Выполните миграцию `migration-multiplayer.sql`
2. Проверьте права доступа к таблицам
3. Убедитесь в корректности схемы

### Проблемы с играми
1. Проверьте логи сервера
2. Убедитесь в корректности данных игроков
3. Проверьте баланс пользователей
