## Общее описание

Данный проект — это игровая платформа для игры в крестики-нолики с возможностью ставок, лидербордом, системой транзакций, админкой и поддержкой мультиплеера. Реализована на Next.js, Supabase, Socket.IO, с разделением на пользовательскую и административную части.

---

## Структура проекта

- **app/** — корневая директория приложения (страницы, layout, глобальные стили, API-роуты, админка, логин и т.д.)
- **components/** — основные UI-компоненты, экраны, админские компоненты, базовые элементы интерфейса
- **hooks/** — пользовательские хуки для работы с мультиплеером, мобильной версией, уведомлениями и т.д.
- **lib/** — вспомогательные функции, типы, работа с Supabase, сокетами, утилиты
- **public/** — статические файлы (иконки, изображения, manifest)
- **styles/** — глобальные стили (Tailwind, CSS)

---

## Пользовательская часть

### Главная страница (app/page.tsx)
- Отображает основной интерфейс игры, навигацию, баланс, лидерборд, профиль, мультиплеер.
- Использует компонент UserInterface.

### Логин (app/login/page.tsx)
- Позволяет войти по email+пароль (Supabase Auth) или username+пароль (кастомный API).
- После успешного входа — редирект в админку (если isAdmin) или на главную.

### Игровой процесс
- Компоненты: game-board.tsx, game-screen.tsx, win-screen.tsx, lobby-screen.tsx, leaderboard-screen.tsx, profile-screen.tsx, home-screen.tsx.
- Поддержка одиночной игры, игры с ботом, мультиплеера через Socket.IO.
- Ставки, банк, определение победителя, возврат при ничьей, начисление выигрыша.

## Админка

### Страницы
- **/admin** — главная страница админки, проверка прав доступа (isAdmin), редирект на /login при ошибке.
- **/admin/dashboard** — дашборд с визуализациями, статистикой, быстрым доступом к разделам.

- **admin-stats.tsx** — статистика по пользователям, играм, транзакциям, объёму, комиссиям.
- **game-results-dashboard.tsx** — аналитика по результатам игр, фильтры, визуализации, экспорт.
- **game-results-table.tsx, game-results-filter.tsx, game-results-pagination.tsx, game-results-visualizations.tsx** — вспомогательные компоненты для отображения и анализа результатов игр.

## API (эндпоинты)

### Пользовательские API (app/api/)
- **/api/auth** — регистрация/логин по username+password (кастомная логика, bcrypt, Supabase).
- **/api/games** — создание, получение списка игр, фильтрация по статусу, пользователю.
- **/api/games/[id]** — детали игры, обновление статуса, завершение, начисление выигрыша.
- **/api/games/[id]/move** — совершение хода, проверка баланса, списание ставки, начисление выигрыша, возврат при ничьей.
- **/api/leaderboard** — получение лидерборда (по выигрышам, по количеству побед).
- **/api/transactions** — создание и получение пользовательских транзакций.
- **/api/users** — получение и обновление данных пользователя.
- **/api/users/[id]** — детали пользователя, обновление профиля.
- **/api/users/[id]/balance** — изменение баланса пользователя (депозит, списание, возврат, проверка на отрицательный баланс).
- **/api/notifications** — получение пользовательских уведомлений.
- **/api/settings** — получение и обновление пользовательских настроек.
- **/api/withdraw_requests** — создание заявки на вывод средств.
- **/api/health** — проверка статуса сервиса.
- **/api/webhook** — обработка внешних платежных вебхуков (депозиты, выводы).
- **/api/cron** — крон-задачи (очистка старых игр, обновление статусов пользователей).

### Административные API (app/api/admin/)
- **/api/admin/games** — просмотр и фильтрация всех игр, детализация.
- **/api/admin/users** — просмотр, фильтрация, поиск пользователей, изменение статуса.
- **/api/admin/users/[id]** — детали пользователя, изменение статуса, просмотр истории.
- **/api/admin/users/[id]/balance** — изменение баланса пользователя админом.
- **/api/admin/notifications** — просмотр и управление уведомлениями.
- **/api/admin/transactions** — просмотр, фильтрация, детализация транзакций.
- **/api/admin/transactions/[id]** — изменение статуса транзакции.
- **/api/admin/stats** — получение сводной статистики по платформе.

## Утилиты и типы

- **lib/types.ts** — все основные типы данных: UserData, GameState, Transaction, Notification, SystemSettings, GameSettings и др.
- **lib/db-actions.ts** — функции для работы с Supabase: создание/обновление пользователя, игры, транзакции, уведомления, получение настроек.
- **lib/socket.ts** — менеджер Socket.IO, синглтон, обработка событий, защита от SSR.
- **lib/supabase.ts** — создание Supabase client (теперь через createPagesBrowserClient), серверный и клиентский режимы.
- **lib/utils/** — вспомогательные функции: работа с датами, форматированием, валидацией, логированием, игровая логика (определение победителя, расчет выигрыша и т.д.).

## Аутентификация и авторизация

- **Supabase Auth (email+password)** — основной способ для пользователей.
- **Кастомный API (username+password)** — альтернативный способ, поддержка старых пользователей.
- **Проверка isAdmin** — для доступа к админке, хранится в таблице users.
- **Сессии** — хранятся в Supabase, localStorage, sessionStorage (очистка при logout).

---

## Работа с играми

- **Создание игры** — через /api/games, списание ставки, проверка баланса, запись в БД.
- **Совершение хода** — через /api/games/[id]/move, проверка очередности, баланса, обновление доски, расчет победителя, начисление выигрыша, возврат при ничьей.
- **Завершение игры** — автоматическое (победа/ничья) или вручную (админка).
- **Мультиплеер** — через Socket.IO, комнаты, события, real-time обновления.

---

## Транзакции и баланс

- **Транзакции** — все изменения баланса фиксируются в таблице transactions (депозит, вывод, ставка, выигрыш, возврат).
- **updateUserBalance** — не позволяет уйти в минус, откатывает действие при ошибке.
- **История транзакций** — доступна пользователю и админу, фильтрация, детализация.

---

## Лидерборд

- **/api/leaderboard** — формируется по сумме выигрышей (win-транзакции) или по количеству побед.
- **Обновляется автоматически** при завершении игр и начислении выигрышей.

---

## Уведомления

- **Пользовательские** — о пополнении, выводе, выигрыше, системные сообщения.
- **Админские** — о новых заявках, ошибках, событиях платформы.
- **/api/notifications, /api/admin/notifications** — получение, изменение статуса, фильтрация.

## Настройки и системные параметры

- **Изменяются только через админку** (для безопасности).
